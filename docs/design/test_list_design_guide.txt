1. 디자인 철학 (Design Philosophy)
----------------------------------
Mellow Wave는 "마음의 파도를 타는 시간"이라는 슬로건 아래,
따뜻하고 밝은 감성을 전달하는 플랫폼입니다.
전체 테스트 페이지는 메인 페이지(index.html)와
완전히 동일한 디자인 시스템을 공유하며,
"깔끔 · 밝음 · 귀여움"의 3대 원칙을 따릅니다.


2. 색상 팔레트 (Color Palette)
--------------------------------
  - Primary:    #FFD166 (따뜻한 노란색)     → 메인 강조, 필터 버튼 active 배경
  - Secondary:  #FF8C94 (소프트 코랄 핑크)  → 검색 버튼, 포인트 컬러
  - Accent:     #FFB366 (잘 익은 오렌지)    → 로고 색상, 카운트 강조
  - Background: #FFFDF5 (따뜻한 화이트)     → 페이지 전체 배경
  - Text Dark:  #4A4A4A (차콜 그레이)       → 기본 텍스트
  - Tag Colors: 각 카테고리별 파스텔 톤
    · 성격: bg-yellow-100, text-yellow-700
    · 연애: bg-red-100, text-red-700
    · 생존: bg-green-100, text-green-700
    · 성향: bg-purple-100, text-purple-700
    · MBTI: bg-pink-100, text-pink-700


3. 타이포그래피 (Typography)
-----------------------------
  - 제목/로고: 'Gaegu' (Google Fonts)
    · 손글씨 느낌으로 친근하고 귀여운 인상
    · 사용처: 헤더 로고, 섹션 제목, 빈 상태 메시지
  - 본문/태그: 'Noto Sans KR' (Google Fonts)
    · 가독성이 높은 산세리프
    · 사용처: 카드 텍스트, 필터 버튼, 설명문

5. 카드 컴포넌트 (Card Component)
-----------------------------------
  구조:
  ┌──────────────────────┐
  │  [그라데이션 배경]    │ h-32, 아이콘 text-5xl
  │      🍰              │ 뱃지: absolute top-2 right-2
  │              [HOT]   │ hover: scale-105
  ├──────────────────────┤
  │  내가 디저트라면?    │ font-bold text-sm
  │  달콤한 디저트로...   │ text-xs text-gray-500
  │  #성격  #MBTI  👤12K │ 태그 + 참여자 수
  └──────────────────────┘

  CSS 클래스:
  - 컨테이너: bg-white rounded-2xl shadow-md overflow-hidden
  - 호버: hover:shadow-xl hover:-translate-y-1
  - 전환: transition-all duration-300
  - 아이콘 영역: bg-gradient-to-br + 카테고리별 파스텔
  - 뱃지: bg-gradient-to-r + text-white text-[10px] font-bold
           px-2 py-0.5 rounded-full
  - 태그: bg-{color}-100 text-{color}-700 text-xs font-bold
          px-2.5 py-0.5 rounded-full
  - 참여자 수: text-xs text-gray-400  (예: 👤 12,345)


6. 필터 버튼 (Filter Button)
-------------------------------
  - 기본 상태:
    bg-white text-gray-500 border border-gray-200
    rounded-full px-5 py-2 text-sm font-bold shadow-sm

  - Active 상태:
    background: linear-gradient(135deg, #FFD166, #FFB366)
    color: #4A4A4A (어두운 글씨로 가독성 확보)
    border-color: transparent
    box-shadow: 0 4px 12px rgba(255, 209, 102, 0.4)

  ※ 이전 버그: active 상태에서 color: white 사용 시
     노란 배경 위에 흰 글씨가 보이지 않는 문제 발생
     → color: var(--text-dark) (#4A4A4A)로 수정하여 해결


7. 참여자 수 집계 (Participant Counter)
-----------------------------------------
  - 각 테스트 카드 하단에 "👤 12,345" 형태로 표시
  - 현재는 testData 배열의 participants 값을 사용 (정적)
  - 추후 Supabase 연동 시 실시간 카운트로 교체 가능
  - 숫자 포맷: toLocaleString()으로 천 단위 콤마 자동 적용


8. 반응형 그리드 (Responsive Grid)
-------------------------------------
  - Mobile   (< 640px):  1열   grid-cols-1
  - Tablet   (640px~):   2열   sm:grid-cols-2
  - Desktop  (768px~):   3열   md:grid-cols-3
  - Wide     (1024px~):  5열   lg:grid-cols-5
  → index.html의 '전체 테스트' 섹션과 동일한 열 배치


9. 물결 애니메이션 (Wave Animation)
--------------------------------------
  main.css에 정의된 3단 물결을 Hero 하단에 배치:
  - wave-1: #FFD166 (노란색), 25초 주기, z-index: 3
  - wave-2: #FFB366 (주황색), 35초 주기, z-index: 2
  - wave-3: #FF8C94 (핑크색), 39초 주기, z-index: 1
  각 물결은 translateX(-50%) 키프레임으로 무한 반복


10. 파일 구조 (File Structure)
---------------------------------
  src/pages/tests/
  ├── test_list.html    ← 전체 테스트 페이지 (본 문서 대상)
  ├── test_list.js      ← 필터링/정렬/렌더링 로직
  └── (각 테스트 폴더)
      ├── index.html     → 테스트 소개 페이지
      ├── question.html  → 질문 페이지
      └── result.html    → 결과 페이지
