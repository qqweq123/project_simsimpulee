# 무인도 생존유형 테스트: 데이터 철학 및 기초 판별 로직 분석 보고서

본 문서는 Mellow Wave의 신규 '무인도 생존유형 테스트'가 내포하고 있는 기초 판별 알고리즘의 무결성을 점검하고, 데이터 거버넌스 및 아키텍처 관점에서 해당 테스트가 플랫폼 생태계에 제공하는 데이터 철학적 가치를 규명하기 위해 작성되었습니다.

## 1. 아키텍처 철학 (Data Philosophy)

해당 테스트는 단순한 흥미 유발을 넘어, 기업의 핵심 데이터 자산(Data Asset)을 수집하고 정제하는 **정교한 감성 텔레메트리 파이프라인(Emotional Telemetry Pipeline)**으로 기능해야 합니다.

*   **단일 진실 공급원(SSOT)의 확립**: 사용자의 모든 선택과 체류 시간은 암호화된 `user_uuid`를 기준으로 `User_Result_DB`와 `Event_Log_DB`에 적재됩니다. 테스트 내에서 발생하는 데이터 결측치나 어뷰징(따닥 클릭, 봇의 난입)은 Supabase Edge의 `SECURITY DEFINER` 게이트키퍼에 의해 원천 차단되며, 이를 통해 마케팅이나 AI 모듈에 제공될 데이터의 100% 무결성을 보장합니다.
*   **컴플라이언스 기반 익명화 (Data Governance)**: PII(개인식별정보)를 별개로 입력받지 않으면서도 브라우저 지문을 통한 추적으로 세그먼트를 묶어냅니다. 이로 인해 개인정보보호법에 저촉되지 않는 가장 클린한 1st-party Data를 합법적으로 획득할 수 있습니다.

## 2. 기초 판별 로직 시스템 (Base Discrimination Logic)

무인도 테스트의 결과 판별 로직은 `calculator.js`와 `data.js`의 객체 매핑 기반으로 구축되어 있습니다.

### 2-1. 단일 책임 매핑 알고리즘 (Single Point Weighting)
10개의 각 문항은 4개의 선택지를 가지며, 각 선택지는 리더(`leader`), 탐험가(`explorer`), 생존가(`survivor`), 외교관(`diplomat`) 중 하나의 타겟 변수(Target Variable)에 1점의 가중치를 배당합니다.

```javascript
// 핵심 환산 로직 (core/calculator.js)
export function getTopType(scores) {
    if (!scores || typeof scores !== 'object') return 'survivor'; // 비정상 접근에 대한 Fallback
    // 순차 탐색을 통한 최빈값 추출 (시간 복잡도 O(N))
    return Object.entries(scores).reduce((a, b) => a[1] >= b[1] ? a : b)[0];
}
```

### 2-2. 로직의 데이터 구조적 한계와 공학적 승리
*   **한계점 (동점자 처리의 단순성)**: 문항이 10개이므로, 확률적으로 `[3, 3, 2, 2]`와 같이 동점이 발생할 확률이 높습니다. 현재 로직(`a[1] >= b[1]`)은 후순위(배열상 앞쪽 객체) 또는 선순위로 편향되는 묵시적 타이브레이커(Tie-breaker)를 가지고 있습니다.

`추가 제언. 특정 문항에 가중치를 부여하는 방식으로 구현하기, 혹은 특정 문제에서 얼마나 깊게 생각했는지에 대한 가중치를 부여하는 방식으로 타임스탬프를 활용한 동점자 처리 로직을 구현을 할 수 있지만, 이는 user나 서버의 리소스를 낭비하는 결과를 초래할 수 있다. 따라서 최대한 동점자가 안나오는 현상을 지향 하여야 한다.`
*구체적인 목적 1. 특정 문항에 가중치를 부여하는 방식으로 동점자를 없애기*
*구체적인 목적 2. 동점자가 나오는 환경을 지양하기*
*구체적인 목적 3. 다만 동점자 자체에 매몰되어서, 테스트의 방향성과 user에서 선사하는 경험, 재미를 해쳐서는 안된다. 그러므로 최대한 재미와 경험을 중시하며 동점자를 판별하는 상황은 후순위이다.*

*   **극강의 처리 효율 (Cost Optimization)**: 이처럼 구조를 단순화함으로써 브라우저 단의 메모리 연산 압박을 0으로 수렴시켰습니다. 수십만 명이 동시 접속하더라도 클라이언트 연산만으로 결과를 도출하기 때문에, 서버리스 컴퓨팅(Edge) 비용 및 DB의 처리량 한계를 근본적으로 우회한 성공적인 '비용 선형화' 모델입니다.

## 3. 데이터 마트 및 타 부서 파이프라인 연계 방안 (Data Enablement)

이 단순해 보이는 10개짜리 이벤트 로그 셋은 추후 엄청난 부가가치를 지닌 Feature Data로 치환됩니다.

### 3-1. 참여형 AI 소설 시스템을 위한 프롬프트 엔지니어링 
수집된 데이터를 바탕으로, 이후 기획된 'AI 기반 참여형 소설 파이프라인'에 투입될 완벽한 시드(Seed) 데이터가 됩니다.
*   **Feature Extraction**: 유저가 무인도에서 [식량 우선(Survivor)]을 골랐는지 [탐험(Explorer)]을 골랐는지에 대한 벡터 맵이 RDBMS에 남습니다.
*   **LLM 파이프라인 (RAG)**: 이 데이터를 `user_uuid` 기준으로 Fetch해와서 GPT-4 등 생성형 AI의 시스템 프롬프트(System Prompt)에 컨텍스트로 주입하면, **"유저 본인의 성향이 100% 반영된 개인화된 텍스트 어드벤처 시나리오"**를 즉각적으로 동적 생성할 수 있습니다.

### 3-2. 추천 엔진용 메타 태깅 (Tag-based Matrix)
*   **교차 추천 (Cross-Selling)**: `island_test`에서 `diplomat(외교관)` 판정을 받은 유저는 기본적으로 이타적인 성향 수치(Social=95)가 높습니다. 따라서 추천 배너 렌더링 시, 잔혹한 성향의 `전생 악마 테스트` 배너를 강제로 노출 시키지 않고(블랙리스트), 이타주의 기반의 `연애 능력치 테스트` 배너를 우선 노출(가중치 +2.0)하여 체류 및 전환율(Bounce Rate 방어)을 체계적으로 높일 수 있습니다.

## 4. 데이터 디렉터 총평 및 개선 권고

무인도 생존유형 테스트는 기획적 흥미로움 뿐만 아니라, 엔지니어링적으로 **'가장 얇고 예리한 칼'**처럼 데이터 캡처의 본질을 완벽히 수행하고 있습니다.
다만, 향후 100만 명 이상의 MAU 확보 시점(Phase 3)을 위해 다음 사항의 병렬 백로그 셋업을 권고합니다.

1.  **동점자 타이브레이커 고도화**: 문항별 가중치를 1점이 아닌 `[1.1, 1.05]` 등 소수점으로 부여하여 확률적 동점을 완전히 분쇄하는 스키마. (현 DB `weights_jsonb` 컬럼만 업데이트하면 코드 수정 없이 즉시 반영 가능)
2.  **데이터 레이크 적재 (Cold Storage)**: 수백만 건의 원시 이벤트 로그는 3개월 후 자동 패키징되어 저렴한 AWS S3 등 콜드 스토리지로 마이그레이션되는 Data Lifecycle Policy 설정 필요.

> **결론**: Mellow Wave 플랫폼의 근간인 '심리 기반 데이터 파이프라인'의 교보재로 삼을 수 있을 만큼 완벽한 기초 형태를 갖추고 있습니다. 데이터 품질과 RLS 보안 정책이 확보되었으므로 마음껏 마케팅 트래픽을 유입시켜도 무방합니다.
