# 05. 프론트엔드 통합 웹 규격서 (Frontend & Asset Standards)

> **목적**: 데이터 아키텍처 뒷단이 아무리 훌륭해도 브라우저 렌더링 단계에서 병목 현상이 생기면 품질이 저하됩니다. 본 문서는 프론트엔드 UI/UX 렌더링, 이미지 에셋 최적화 및 레이아웃을 다루는 최종 클라이언트 기준 규격입니다.

---

## 1. 이미지 에셋 파이프라인 (Asset Data Pipeline)

디자인팀에서 생산되는 원본 그래픽이 서비스 생태계에 반영되기까지의 규격입니다.

1. **절대적 압축 포맷 (WebP Mandatory)**
   * UI 에셋, 썸네일, 배너, 일러스트레이션 등 모든 비트맵 그래픽 레이어는 브라우저 로딩 시 **무조건 `.webp` 형식**으로 제공되어야 합니다. PNG/JPG로 서비스되는 것은 허용되지 않습니다.
   * *예외 규정*: 카카오톡 공유 등 구형 모바일 크롤러 대응용 Open Graph 이미지(`og:image`)에 한해서 보수적인 호환성을 위해 압축된 `.jpg` 형식을 병행 사용합니다.

2. **심리스(Seamless) 텍스처 패턴 타일링**
   * 배경 화면을 통짜 고해상도 이미지 파일로 덮어씌우는 방식(Legacy)을 금지합니다.
   * 상하좌우 반복이 가능한 패턴 조각 타일(Tile)을 생성하여 `background-repeat: repeat` 와 `mix-blend-mode: multiply` CSS 속성 등을 결합하고, 코드 단(Layered CSS gradients)에서 색감을 섞는 하이브리드 렌더링으로 통신 비용을 최소화합니다.

3. **자동화된 캔버스 전처리 (AI Garbage Cleansing)**
   * AI 생성 일러스트의 외곽에 붙어 나오는 임의의 회색/반투명 여백 프레임을 CSS(`object-cover` 등) 필터로 가리는 방식은 DOM 렌더링 비용을 가중하므로 불합격 처리합니다.
   * `sharp` 등의 파이프라인 자동화 툴봇(Node.js Script)을 통해 DB/CDN 적재 전 원본 데이터의 외곽 여백 선을 잘라내는 크롭 전처리를 물리적으로 수행해야 합니다.

## 2. 프로모션 배너 (V9 Hybrid) 디자인 가이드

사용자에게 추천 테스트를 띄우기 위한 슬라이드/목록형 배너의 황금비율 레이아웃 표준 규격입니다. 이터레이션을 거쳐 확정된 'V9 하이브리드 풀-블리드' 체제를 따릅니다.

1. **배너 해상도 표준 (Standard Resolution)**
   * 이미지 에셋 해상도는 **1200 x 330 픽셀 (가로:세로 약 4:1의 와이드 파노라마 비율)** 로 엄격히 통제하여 업로드합니다.

2. **하이브리드 풀-블리드 (Hybrid Full-Bleed)**
   * 모바일 스크린 환경에서, 내부 콘텐츠와 구별감을 주기 위해 상위 컨테이너의 패딩(Padding)을 좌우로 강제 취소하는 **네거티브 마진(`margin-left: -Xpx`, `margin-right: -Xpx`)** 을 활용합니다. 화면 좌우 모서리에 틈 없이 꽉 차게 확장되는 레이아웃입니다.
   * 단, 풀-블리드로 화면을 덮되 이 개체가 스크롤 가능한 독립 버튼(카드)임을 인지시키기 위해 이미지 모서리는 약간 둥글려진 캡슐 형태(`border-radius` 적용)를 엄수해야 합니다.

3. **가독성 밸런스 유지**
   * 배너 내의 글래스모피즘(투명) 텍스처나 타이포그래피는 주변 백그라운드 노이즈와 대비율(Signal-to-Noise Ratio)을 지켜야 합니다. 필요한 경우 안쪽 그림자나 미세한 여백을 허용하여 가독성을 보정해야 합니다.
