본 기획서는 island테스트 구현 로직과 차후 템플릿 화를 하기 위해 쓰여진 기초 개념도 입니다.

**테스트 구현 순서**

테스트는 아이디어 탐색 질문지가 7개 이하인 lite test, 13개 이하인 regular test, 13개 이상인 deep test로 나뉜다. (이 때에 구체적인 콘셉 방향성 까지 선정하기. 공포, 개그, 힐링, 이야기, 소설, 자극적, 공감 등) -> 아이디어에 맞는 기획서 작성 -> 질문지와 결과지 초안 작성 -> 알맞은 템플릿 탐색 -> 질문지와 결과지 정교화 -> 배경, 결과이미지, index.html 및 mallow wave 메인화면 및 전체 테스트에 에 표기될 이미지 생성 *단 캐릭터 일관성을 유지 하여야 하며, mallow wave메인 화면에 표기될 이미지는 테스트 전체의 콘셉을 함축적으로 보여줄 수 있는 이미지를 사용하거나, 심리테스트의 도입부 등을 표현하며 user가 참여하고 싶은 이미지를 만드는 것을 목적으로 한다.* ->템플릿을 기반으로 한 변형을 통해 결과페이지 구현 혹은 테스트 분위기와 맞는 고유 결과페이지 생성 
-> 결과 페이지의 톤앤매너를 반영하여 전체 테스트 구현 작업 진행 -> 테스트 전체적 검토 감성, 이슈성, 적합성, 톤앤매너, 데이터 무결성 등 -> supabase에 테스트 데이터 업로드 -> 최종배포
